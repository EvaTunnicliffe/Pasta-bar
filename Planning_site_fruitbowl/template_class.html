Brief

This python program is a pasta bar ordering system. It is designed to help users to easily place an online order and users can either pick up or have it delivered to their homes. The menu that we have used is from the pasta bar in Wellington, 1154. I have created this program using pycharm.

Describing Relivent Implications

  • Usability
    • Usability is about how a digital product should be easy to operate. The ordering system should communicate to the user what is going on, and what the need to do for it to work correctly. For example, they could use words, the way the screen is organised, or even colour changes to help the user to understand what is going on. As well as this, if the user inputs something incorrectly, it should be clearly and politely communicated that they are entering an incorrect input. If they entered something that is too large or too small, they should be notified, as well as if they enter an integer per say, when the program was expecting a string. The program needs to have an error prevention function, and when something goes wrong, the program should not crash and the user should get clear messages on how they can fix things. With polite and clear instructions, the user can intuitively learn how to navigate the program quickly. Usability means that all options are easy to understand, and that all actions are easy for the user to take.

  • Functionality
    • One of the most important thing about this program Is that is has to be created in a way that it is functional. This means making sure that it does not crash, whatever the input is, and all of the arithmetic is correct. This includes making sure the total price of the order is correct, with no extra money added on for no reason. It also means ensuring that all variables and lists are all reset and set back to zero once an order is complete, so that the next order starts on a fresh page.

  • Sustainability and Future-proofing
    • Another very important step to creating a program is to make sure that it is future-proof. As things evolve during life, we need to be able to keep updating our program. Our program needs to be able to change with the times which means making sure that it can be easily updated and you can add new features to it easily, along with ensuring that the program is able to be maintained. The program needs to be commented so that anyone who opens it can see what is going on and they are able to see where everything is. The program also needs to be structured in a way that new features can be added easily without impacting the rest of the program. The program should be able to be handed over to a different developer and they should be able to understand it quickly without any difficulties.

Addressing Relivent Implications

  • Usability
    • Usability basically means that the program is not going to crash, and that it is easy to understand. When doing my code, I made sure that every time the user had to enter something, their input was validated. This meant that whatever the user entered, it wouldn’t cause any errors, but instead direct them on what they did wrong, and ask them to enter their input again. For example, if the program was expecting an integer to be entered but the user inputted a string, it would tell them what the problem was in a polite way, and ask them to re-enter. Everything is case insensitive, meaning if they type capitals or lowercase letters, it won’t crash the system, it will just workk like normal. This allows the user to go through the program smoothly without any confusion. Another thing about usability is that it should be easy to understand. How I have done that is by using a very simple menu design, one that all users are familiar with. It’s like a Home Screen and is easy to navigate.

  • Functionality
    • There are a few ways that I have made sure that this program is functional and that ensures that everything works correctly and smoothly. One of the things that I did was test my program throughout the development process and used systematic testing. This means that if there were any issues at any point, I was able to solve them before I moved on, ensuring all of the problems were fixed by the time I reached the end. When working on different validation sections, I pulled them out and worked on them in our testing page. This meant that I was able to work on these pieces of code without worrying about anything else in the code crashing. At the end of each sprint, I went through the whole program, entering inputs that I knew weren’t the ideal ones to see what would happen. This helped me to recover any problems that occurred, and I was able to fix them. I used everyones feedback to ensure that their experience was the best it could possibly be, even if it just meant adding breaks in my code so it became easier to read and understand. Doing all of these tests means that the end result is error free, and whatever the customer puts in, it will all work as it is supposed to and not crash. I got people to not follow the instructions to ensure that all my validation was working correctly and had clear instructions. Once I had finished, I got two different people to fully run through my code, one being Sophie, and one being my dad. These two people were unfamiliar with my code, and didn’t understand how it worked. It was important for me to know that someone who had never seen it before could still sue it and understand it without crashing.

  • Sustainability and Future-proofing
    • One of the most important parts of my program is that I have made it future-proof. This means that in the future, if I were to ever come back to the program, or if I handed it over to another developer, it will be very easy for them. One way that I have done this is my commenting on anything I thought would be useful to know. I explained when I had put new loops or function, and I also commented on what each little part of the code was for so that if anyone were trying to fix or update my program, they would know exactly what was going on. I have also structured it using a function based build. This means that for each new feature, there is a new function, which means there is no impact on what is already there. It is easy to add new items to the menu, and add new code to the program. I made sure that the only thing that needed to be updated is the main menu function, if something needed to be added or even deleted. I have doctype the program as well which helps people to clearly understand what was going on in the program. I also completed a pop 8 check that was attached through the program which made sure my code was correctly formatted according to the python conventions. That made sure that if things were incorrect, you could see what needed to be changed. This made it easy to understand what wasn’t right. For example, if there were too many white spaces at the end of the line, it would call you out on that. This is how I learnt about the different ways python likes to function, and how specific they are when it comes to little things like this.
